# 1) 사용자 여정(User Journey)

## 핵심 사용자: “면접 끝나고 멘탈 흔들리는 구직자”

- 목표: **탈락/불안 → 다음 면접에서 개선되는 행동 1개 확보**
- 제약: **타이핑을 오래 못 함**, 기억이 흐려짐, 피드백이 없어서 방향을 잃음

---

## Journey A. 면접 직후(가장 중요한 루프)

### A1. 홈 → “면접 복기 입력(간단)”

- **간단 입력(기본값)**: 1~2분 내 완료 (선택지 기반)
    - 회사/직무/날짜/분위기(1~5)
    - 주요 질문 1~3개(선택 + 한 줄)
    - 답변 체감(1~5)
    - 컨디션 조절 방법(복수선택) + 효과(1~5)
- (선택) **상세 입력**: 메모/질문 추가/한 줄 회고

### A2. 저장 직후 팝업 → “추천 행동(체크박스)”

- 저장 완료 즉시 팝업 노출(1회)
- 내용은 딱 3줄:
    1. 오늘의 핵심 요약 1줄
    2. **Patch 미션 1개**(면접 기반) → 저장
    3. **멘탈 미션 1개**(비의료 행동 기반) → 휘발
- 버튼: `미션 저장` / `닫기`

### A3. 피드백 페이지(자동 생성)

- 저장 후 자동으로 피드백 리포트 생성
- 구성:
    - 전반 요약(짧게)
    - 감정 요약
    - 개선 포인트 1개(가장 큰 것 1개)
    - 다음 전략 1개
    - 미션 1개(Patch) + 멘탈 미션 1개
    - 체크리스트(부족 태그 기반)
    - 
    - **요약(3~5줄)**: 이번 면접의 흐름/막힌 지점/전달 방식 요약
    - **개선 1개(1문장)**: 가장 영향 큰 결함 1개만 고정
    - **체크리스트(3~7개)**: 실행 가능한 준비 항목(짧은 문장)

---

## Journey B. 누적 분석(가치가 쌓이는 루프)

### B1. 마이페이지 → 통계 확인

- 반복된 실수 TOP3
- 자주 나오는 질문 TOP5(카테고리별)
- 감정/긴장 추이(최근 5회 평균)
- 체크리스트(면접 전/후 루틴)

### B2. 과거 면접 상세 → 피드백 재확인

- 유사 회사/직무 면접 전에 과거 기록을 다시 보고 준비
- (선택) PDF 저장/공유/인쇄는 MVP에선 옵션(후순위)

---

## Journey C. 공고 기반 성장(다음 단계, 겹치지 않게 설계)

### C1. 기업 공고 입력 → 요구 역량 추출

- 회사/직무/마감/면접일 + 공고 텍스트(또는 링크)
- 저장하면 요구 역량 태그 생성

### C2. 공고 상세 → “Build 미션 1개”

- 공고 요구 역량 vs 내 약점/증거 부족을 비교해 **Build 미션 1개만** 제공
- Build 미션은 “답변”이 아니라 “증거(프로젝트/산출물)” 생성형
    - 예: 테스트 추가, README 보강, 배포 링크 만들기 등

> **차별점 고정**
> 
- 면접 미션 = **Patch(답변/행동 교정, 15~30분)**
- 공고 미션 = **Build(증거/프로젝트 산출, 3~6시간+)**

# 2) PRD (Product Requirements Document)

## 2.1 제품 개요

- 제품명(가칭): **SkillSeeker – Interview Review**
- 한 줄 소개: **면접 직후 2분 복기 → 자동 피드백 & 다음 행동 1개를 주는 면접 학습 플랫폼**
- 핵심 가치:
    1. 입력 부담 최소(선택지 기반)
    2. 불확실성 감소(“다음에 뭘 고칠지 1개” 고정)
    3. 누적 데이터 기반 성장(반복 실수/질문 통계)

---

## 2.2 문제 정의(Problem)

- 구직자는 면접 후 **탈락 사유를 모르고**, 무엇을 고쳐야 할지 불명확함
- 복기를 하더라도 **메모 수준**에서 끝나 누적/개선으로 이어지지 않음
- 멘탈이 흔들리는데, 이를 **행동(루틴/미션)**으로 바꾸는 장치가 부족함

---

## 2.3 목표(Goals)

### MVP Goals

1. 사용자가 **면접 복기를 빠르게 기록**할 수 있다(2~5분)
2. 기록을 기반으로 **피드백 리포트**를 즉시 받을 수 있다
3. 반복 패턴을 보여주는 **통계(실수/질문)**를 제공한다
4. 면접 직후 **추천 행동/미션**으로 다음 액션을 고정한다
5. 공고 입력 시 **Build 미션**을 제공한다(기본 1개)

### Non-goals(이번 MVP에서 안 함)

- 면접 **녹음 업로드/자동 전사**
- 의료적 조언(약물 권장/효능 판단)
- 기업용 관리자 대시보드/채용 연동

---

## 2.4 성공 지표(Success Metrics)

- Activation: 가입 후 **첫 면접 복기 작성률**
- Speed: 면접 복기 평균 작성 시간(목표: 2~5분)
- Value: 피드백 페이지 도달률 / 미션 저장률
- Retention: 7일 내 재방문률(통계 확인/두 번째 복기 작성)
- Quality(정성): “다음에 뭘 할지 명확해졌다” 만족도

---

## 2.5 사용자/페르소나

- P1: 취준생(기술/일반 면접 반복), 면접이 잦고 멘탈 소모 큼
- P2: 이직 준비 경력자, 면접 질문 패턴을 빠르게 정리하고 싶음

---

## 2.6 정보구조(IA) / 화면 구성

### Top Nav(상단 메뉴)

- 홈 / 피드백(기록) / 기업(공고) / 마이페이지

### 화면 목록(MVP)

1. **홈 및 면접 복기 입력(간단/상세 토글)**
2. **면접 상세 + 피드백 리포트**
3. **면접 기록 목록(이전 기록)**
4. **기업 공고 입력**
5. **기업 공고 목록/상세(미션 포함)**
6. **마이페이지(통계/체크리스트/멘탈 요약)**
- (공통) 저장 직후 **팝업(추천 행동)**

# 3) 기능 요구사항(Functional Requirements)

## 3.1 홈(Home)

**요구**

- 오늘의 미션 카드(있으면 1개만)

**수용 기준(AC)**

- 사용자는 홈 및 복기 입력 화면에 진입한다.

## 3.1.1 면접 복기 입력(Quick/Detail)

### 입력 항목

**기본 정보**

- 면접 날짜(default=오늘)
- 회사명(텍스트)
- 지원 직무(텍스트/선택)
- 면접 분위기(1~5)

**면접 내용**

- 면접 내용 요약
- 주요 질문 목록(1~3개 기본, 추가 가능)
    - 버튼 눌러서 꼬리질문 추가 가능

**면접 체감도**

- 면접 완성도(1~5)
- 한 줄 회고(선택)

**컨디션 조절 기록**

- 컨디션 조절 방법(복수선택) nullable
- 면접 긴장완화도 (효과)(1~5)

**UX**

- 기본은 **간단 입력**, 필요 시 “상세 입력 펼치기”
- 타이핑 최소화(칩/슬라이더/체크박스)

**수용 기준(AC)**

- 사용자는 2분 내 최소 입력으로 저장 가능하다(필수만).
- 저장 시 “면접 상세/피드백”으로 이동한다.
- 저장 직후 팝업이 1회 표시된다. - 면접 분위기, 긴장도 조지면 응원 메시지 팝업

---

## 3.3 피드백 리포트(면접 상세 내 섹션)

**출력 구성**

- 전반 요약(3-5줄) - text
- 향후 개선 사항 2-3개(부족한 점): 미흡사항 및 태도 - text
- 향후 면접 준비 전략 - text
- 체크리스트 (면접 내: 단기)
- 체크리스트(부족 태그 기반: 장기, 스펙관련)

**중요 원칙**

- 개선사항은 2-3개
- 체크리스트는 각 항목(단기/장기) 당 2-3개
- LLM이 “판단”하지 않고, 입력(선택지/태그/빈도)을 기반으로 정리/추천

**수용 기준(AC)**

- 미션을 저장/완료 처리할 수 있다(상태 변화).

---

## 3.4 과거 면접 기록(생성/목록/조회/삭제)

**요구**

- 면접 기록 생성
- 면접 기록 수정
- 면접 기록 목록 조회
- 면접 상세 조회(→ 피드백 페이지로 연결 가능)
- 면접 기록 삭제

**수용 기준(AC)**

- 사용자는 과거 기록을 날짜/회사로 쉽게 찾는다.

삭제 시 관련 피드백/미션도 함께 제거된다.

---

## 3.5 마이페이지(통계/멘탈/체크리스트)

**통계**

- 반복된 실수 TOP3(키워드+횟수)
- 자주 나오는 질문 TOP5(카테고리)
- 감정 추이(최근 N회 평균)

**멘탈 관리 정보(축약)**

- “개인 기록” 수준만: 방법/타이밍/체감(좋음/보통/별로)
- **의료적 권장/판단 문구 금지**

**체크리스트**

- 면접 전 루틴 / 면접 후 루틴

**수용 기준(AC)**

- 3회 이상 기록이 쌓이면 TOP 항목이 자동 계산되어 표시된다.
- 멘탈 탭은 “내 기록 요약”만 제공하고 권장/진단을 하지 않는다.

---

## 3.6 기업 공고 입력/목록/미션(Build)

**입력**

- 회사명
- 직무
- 마감/면접일(선택)
- 공고 텍스트(또는 링크)
- (선택) 회사 개요/위치/모집인원

**출력**

- 요구 역량 태그 목록
- **Build 미션 1개**(프로젝트/역량 향상, 증거 생성형)
- “스펙 체크리스트”(선택)

**수용 기준(AC)**

- 사용자는 공고 1개를 저장하고 목록에서 조회할 수 있다.
- 공고 상세에서 Build 미션 1개가 생성된다(없으면 이유 표시: “정보 부족”).

---

## 3.7 팝업(면접 이후 추천 행동)

**요구**

- 면접 저장 직후 1회 노출
- “추천 행동/미션” 체크박스(또는 저장 버튼)

**수용 기준(AC)**

- 팝업에서 미션 저장이 가능하며, 저장하면 홈/마이에서 추적된다.

---

# 4) 데이터 모델(간단)

- User
- Interview
    - company, role, date, 분위기, 완성도, 회고, 메모
- InterviewQuestion
    - interviewId, category, questionText, answerRating(1~5)
- InterviewEmotion
    - interviewId, emotionTag[], intensity(1~5), symptoms(optional)
- ConditionLog
    - interviewId, methods[], effect(1~5), memo(optional)
- FeedbackReport
    - interviewId, summary, improvementOne, strategyOne
- Mission
    - type(PATCH/MENTAL/BUILD), source(interview/job), title, description, status
- JobPosting
    - company, role, deadline, interviewDate, text, extractedSkillTags[]

---

# 5) 생성/추천 로직(애매함 줄이는 규칙)

- **LLM은 “정리/요약/태깅” 역할**
- “좋고 나쁨 판정”은 최소화하고,
    - 반복 태그 빈도
    - 낮은 답변 체감(1~2)
    - 감정 강도(4~5)
        
        같은 **관측치 기반 규칙**으로 미션 1개를 고른다.
        

---

# 6) 보안/정책(최소 가드레일)

- 사용자가 입력한 “개인적 보조 수단(의료/약물…)”은
    - 개인 기록/회고로만 저장
    - 앱은 권장/진단/효능 판단을 하지 않음
- 데이터 삭제 기능 제공(본인 통제)
